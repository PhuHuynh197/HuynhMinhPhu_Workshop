In this section, I would like to share my personal feedback about my internship experience in the **Workforce Bootcamp - First Cloud AI Journey** program at **Amazon Web Services Viet Nam Company Limited**. The internship gave me the opportunity to learn AWS services through hands-on labs, write technical blogs, and develop the final project named **PharmaCare AI**.

Overall, this internship was a meaningful learning journey for me. It helped me understand cloud computing more practically, especially how AWS services can be combined to design and deploy a real system.

### Overall Evaluation

**1. Working Environment**  

The working and learning environment during the First Cloud AI Journey program was open, friendly, and suitable for students who wanted to learn cloud computing from the beginning. Although the program required self-learning and continuous practice, the learning path was clear and helped me know what to focus on each week.

The environment encouraged me to explore AWS services, ask questions, and improve my technical knowledge step by step. Through the workshops and lab activities, I became more familiar with how cloud systems are built and managed in practice.

One thing I appreciated was that the program did not only focus on theory. Instead, it allowed me to practice directly with AWS services such as IAM, VPC, EC2, RDS, CloudWatch, AWS Backup, ECS, and other related services. This helped me understand the differences between reading documentation and actually working with cloud resources.

**2. Support from Mentor / Team Admin**  

The mentor and team admin provided useful guidance throughout the internship. When I had difficulties understanding AWS services or organizing my report content, the guidance helped me know how to continue and improve my work.

I also appreciated that the mentor did not only provide direct answers, but also encouraged self-learning and problem-solving. This helped me become more independent when working with AWS labs, troubleshooting errors, and documenting my work.

The team admin also supported the learning process by providing necessary information, schedules, and instructions. This made the internship process more organized and easier to follow.

**3. Relevance of Work to Academic Major**  

The internship content was highly relevant to my major in **Information Security**. Many AWS topics were connected to security concepts such as identity and access management, network security, database protection, monitoring, backup, and secure system design.

For example, when learning IAM, I understood more about permission control and the principle of least privilege. When learning VPC and Security Groups, I understood how network access is controlled in a cloud environment. When working on PharmaCare AI, I had to consider authentication, authorization, database security, Secrets Manager, WAF, monitoring, and backup.

This helped me see the connection between university knowledge and real-world cloud security practices.

**4. Learning and Skill Development Opportunities**  

During the internship, I had many opportunities to improve both technical skills and documentation skills.

In the first seven weeks, I learned fundamental AWS services through AWS Study Group labs. These labs helped me understand the basic building blocks of cloud infrastructure, including compute, networking, database, monitoring, backup, and security.

From week 8 to week 12, I worked on the **PharmaCare AI** project. Through this project, I learned how to analyze requirements, design an AWS architecture, choose appropriate services, design a database, and document the deployment process.

Besides technical skills, I also improved my ability to write worklogs, technical blogs, project proposals, and workshop documentation. This was important because a cloud engineer not only needs to build systems, but also needs to explain architecture and document implementation steps clearly.

**5. Culture and Team Spirit**  

The program created a positive learning culture. Everyone was encouraged to learn, practice, ask questions, and share knowledge. This made me feel more motivated during the internship.

Although some AWS topics were difficult at first, especially VPC, RDS, serverless architecture, and AI/RAG-related services, the learning environment helped me continue exploring instead of giving up. The program also gave me a better habit of researching documentation and reviewing AWS best practices.

The team spirit was shown through the way members supported each other during labs, shared experiences, and discussed technical issues. This helped me feel that I was part of a learning community, not only an individual student completing a report.

**6. Internship Support and Learning Resources**  

The program provided a clear learning direction through AWS Study Group workshops and hands-on labs. These resources were very useful because they allowed me to learn AWS services in a structured order, from basic services to more advanced topics.

The learning resources helped me gradually build a foundation before starting the final project. For example, IAM, VPC, EC2, RDS, Backup, CloudWatch, and Budget labs were very useful when I later designed the PharmaCare AI architecture.

However, I think it would be helpful if the program provided more warnings or notes for labs that may generate higher AWS costs, such as NAT Gateway, Load Balancer, ECS, Security Hub, Transit Gateway, or database-related resources. This would help interns manage AWS credits more safely during practice.

### Additional Questions

**What did I find most satisfying during the internship?**  

The thing I found most satisfying was that I could apply what I learned from AWS labs to my final project **PharmaCare AI**. At the beginning, I only learned services separately, such as IAM, VPC, EC2, RDS, and CloudWatch. Later, I gradually understood how these services can be connected together to build a complete system architecture.

I was also satisfied with the process of designing the PharmaCare AI architecture. The project helped me understand how a real cloud system may include frontend hosting, authentication, API layer, database, AI chatbot, monitoring, backup, and security components.

**What should be improved for future interns?**  

In my opinion, the program could improve by adding more guided sessions for advanced topics such as Amazon Cognito, API Gateway, AWS Lambda, Amazon Bedrock, Amazon OpenSearch Service, and RAG architecture. These services are very useful for modern cloud projects, but they can be difficult for beginners to understand without detailed examples.

The program could also provide a clearer cost-control checklist before each lab. For example, each lab could include a short note about which services may generate cost and which resources must be deleted after completion.

In addition, it would be useful to have more sample project structures or report templates at the beginning of the internship. This would help interns organize Worklog, Proposal, Project, Workshop, Blogs, Self-Assessment, and Feedback sections more efficiently.

**Would I recommend this internship program to my friends? Why?**  

Yes, I would recommend this internship program to friends who are interested in cloud computing, AWS, system architecture, or information security.

The reason is that the program provides a practical learning path. Instead of only studying theory, interns can practice directly with AWS services and understand how cloud infrastructure works. The program is also suitable for students who want to build a foundation before working on real cloud projects.

However, I would also remind my friends that they need to be proactive. AWS has many services, and some topics can be difficult at first. To learn effectively, interns should spend time reading documentation, practicing labs, taking notes, and reviewing what they have done.

---

### Suggestions and Expectations

To improve the internship experience, I would like to suggest a few points.

First, the program could provide more detailed guidance for the final project phase. For example, it would be helpful to have reference architectures for common project types such as serverless web applications, AI chatbot systems, e-commerce systems, or database-driven applications.

Second, the program could include more sharing sessions from mentors or previous interns. These sessions could focus on real project experience, AWS best practices, cost optimization, common mistakes, and career orientation in cloud computing.

Third, I think it would be useful to have more short review sessions after difficult labs. These sessions could help interns summarize what they learned and understand how each service is used in real systems.

Fourth, I would like the program to provide more practical examples about security and cost control. Since cloud resources can affect both security and budget, these topics are very important for students working with AWS credits.

In the future, I would like to continue learning AWS and improving the PharmaCare AI project. I want to gain more hands-on experience with serverless services, AI/RAG architecture, database security, monitoring, and cloud deployment.

Overall, I am grateful for the opportunity to participate in the First Cloud AI Journey program. The internship helped me gain practical AWS knowledge, improve my technical documentation skills, and better understand the role of cloud architecture in real-world systems.
