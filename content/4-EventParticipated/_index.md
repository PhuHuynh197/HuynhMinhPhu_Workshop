## 1. Event: "AWS Community Day"

* **Date:** 09:00, June 06, 2026
* **Location:** Floor 26, Bitexco Financial Tower
* **Role:** Attendee

### Event Description

AWS Community Day was a professional technology event for the AWS and cloud computing community. The event focused on practical knowledge in cloud architecture, modern application deployment, serverless architecture, cybersecurity, artificial intelligence, and career orientation.

The sessions were valuable because they did not only explain theoretical concepts. The speakers also shared real operational experience, common challenges when deploying systems on AWS, and practical lessons from enterprise environments.

This event was closely related to my internship project **PharmaCare AI**, because my project also uses AWS services such as Lambda, API Gateway, S3, CloudFront, RDS, Cognito, WAF, CloudWatch, and AI-related services.

### Key Takeaways & Accumulated Skills

* **Cloud Architecture & Serverless Mindset:**  
  I learned how cloud systems are designed in real environments, especially how AWS Lambda and managed services can reduce server management work. I also understood that serverless architecture still requires careful design for state management, database queries, latency, and cost control.

* **Virtual Machines vs. Containers:**  
  I gained a clearer understanding of the difference between Virtual Machines and Containers. Virtual Machines provide stronger isolation but consume more resources, while Containers are lighter, faster to start, and more suitable for modern application deployment.

* **AI-Powered Security:**  
  The topic about combining **AWS WAF** and **Machine Learning** helped me understand how AI can support cybersecurity. Instead of relying only on static WAF rules, Machine Learning can help detect abnormal network behavior and improve attack detection.

* **DevOps & Operational Mindset:**  
  The career sharing session helped me understand that DevOps is not only about tools. It is also about automation, monitoring, reliability, incident handling, and continuous improvement.

* **Career Orientation:**  
  The event gave me a clearer view of the career path from Helpdesk or Sysadmin to DevOps and Cloud Engineer. It motivated me to continue improving my skills in AWS, Linux, networking, automation, and cybersecurity.

### Application to My Internship Project

The knowledge from this event was useful for improving my **PharmaCare AI** project.

First, the serverless architecture discussion helped me design AWS Lambda backend functions more carefully. In my project, Lambda is used as the backend API layer, so I need to keep the logic clean, optimize database queries, and avoid unnecessary processing.

Second, the AWS WAF topic helped me understand why WAF should be attached to CloudFront to protect the public web layer from abnormal requests.

Third, the operational mindset from the event supported the final phase of my project, where I configured CloudWatch Logs, CloudWatch Alarms, SNS notification, AWS Backup, and IAM least privilege.
