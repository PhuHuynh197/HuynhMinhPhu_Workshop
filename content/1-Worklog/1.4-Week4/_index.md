### Week 4 Objectives

* Understand the basic concepts of Amazon Elastic Compute Cloud (Amazon EC2).
* Learn how to launch and manage Linux and Windows EC2 instances.
* Practice connecting to EC2 instances using SSH and Remote Desktop Protocol (RDP).
* Learn how Security Groups control inbound and outbound traffic for EC2.
* Understand Amazon EBS, Snapshot, AMI, and instance recovery concepts.
* Study the basic workflow of VM Import/Export and how virtual machines can be migrated to AWS.

### Week 4 Achievements

* Understood the role of Amazon EC2 in providing resizable virtual servers on AWS.

* Learned the main components required to launch an EC2 instance, including:

  * Amazon Machine Image
  * Instance type
  * Key pair
  * Security Group
  * Public IP address
  * Private IP address
  * VPC and subnet

* Practiced launching and connecting to a Windows EC2 instance using Remote Desktop Protocol.

* Practiced launching and connecting to an Amazon Linux EC2 instance using SSH.

* Understood how Security Groups protect EC2 instances by controlling inbound and outbound traffic.

* Learned the basic difference between Linux and Windows EC2 connection methods.

* Understood how Amazon EBS provides persistent block storage for EC2 instances.

* Practiced creating EBS Snapshots for backup purposes.

* Learned how a custom AMI can be created from an existing EC2 instance.

* Practiced launching a new EC2 instance from a custom AMI.

* Understood the basic deployment flow of a web application on EC2.

* Learned the basic concept of VM Import/Export and how it supports migration from an on-premises virtualization environment to AWS.

* Cleaned up temporary EC2 resources after the lab to reduce unnecessary AWS credit usage.

* Completed week 4 with a stronger understanding of AWS compute services and virtual machine management.
